Interactive visualization of genome phylogenies

JScandy is a fully interactive tool to allow visualisation of a phylogenetic tree and associated metadata and genomic information such as recombination blocks, gene-presence/absence or SNP variation.

Running is as simple as loading this page in Google Chrome and then dragging on your data. That's it.

Data that you can visualise

  1. Phylogenetic tree is the backbone of the visualization as it links together all the other data. Trees must be in Newick format (which is the standard output from software such as RAxML). If you need to convert your tree to a different format try using FigTree.
  2. Metadata can be displayed alongside the tree. A simple csv file with the first column containing sample names which match the tips of the tree is used.
  3. Genome annotation file can be read in gff3 format. If you need to convert an embl file to a gff3 file try using Artemis or install seqret and convert via seqret -sequence GFF_FILE -feature -fformat embl -fopenfile EMBL_FILE -osformat gff -auto
  4. Genomic data (the main bit!) JScandy is designed to visualise the output of three pieces of software—Gubbins, Roary or called SNPs. We are working on increasing the files that JScandy can display, however any output which uses the same format should (in theory) work fine.
    • Gubbins (gff3 format—each line links a whitespace seperated list of taxa with genomic co-ordinates). If you have old output from gubbins (a _rec.tab file) then there is a simple python script which will convert this for you.
    • Roary (gene_presence_absence.csv)
    • SNP data COMING SOON
    • Plot data (GWAS) COMING SOON
