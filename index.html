Interactive visualization of genome phylogenies

JScandy is a fully interactive tool to allow visualisation of a phylogenetic tree and associated metadata and genomic information such as recombination blocks, gene-presence/absence or SNP variation.

Running is as simple as loading this page in Google Chrome and then dragging on your data. That's it.

Data that you can visualise

  1. A tree is the backbone of the visualization as it links together all the other data. Any tree in Newick format will do which is the standard output from software such as RAxML
  2. Metadata can be displayed alongside the tree. A simple csv file with the first column containing sample names which match the tips of the tree is used.
  3. Genome annotation file can be read in gff (v3) format.
  4. Genomic data (the main bit!) JScandy is designed to visualise the output of three pieces of software—Gubbins, Roary or called SNPs. However any output which is in the following formats should work...
    • GFF format (Gubbins). Each line links a whitespace seperated list of taxa with genomic co-ordinates.
    • CSV (Roary) COMING SOON
    • VCF (SNPs) COMING SOON

Install your own (local) copy

JSCandy is entirely used from within the browser and runs entirely on your computer (i.e. entirely client side). You can either load the page at http://pneumogen.staging.sanger.ac.uk/jscandy (the easiest option) or download the project from github and have a local copy on your computer. Here are some tips on installing your own local copy:

  1. Node and Node Package manager (npm) are needed to build the javascript bundles. On OS X I recommend following this guide which uses homebrew and nvm.
  2. Clone the github repo (what is cloning?) or download the zip files via github.com/jameshadfield/JScandy
  3. install the required packages by running npm install in the JScandy directory.
  4. run npm run build to build the javascript files into a single bundle.
  5. open index.html in google chrome
